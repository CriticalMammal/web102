Advanced CSS Selectors

A Few More Selectors...

So far, we've learned how to target our HTML using basic selectors. This includes targeting elements via their HTML tags as well as using ID's and classes. In the following few examples, we're going to learn two new and very useful selectors, the direct child selector and the immediate adjacent sibling selector.

Direct Child Selector  >

The direct child selector allows us to target the immediate child elements of the same type nested in a parent element. The selector is simply a right chevron arrow, or >

In the following example, I have used the following CSS rule to color all of the first-level, or direct-child li elements red. You can think of the li elements one level deep (nested) as the direct children li elements of the ul element. Here is the CSS:

ul > li {
    color: red;
}

So, first I am targetting the ul element, and then I am using the direct child selector to target all of the direct child li elements of this ul element. Note: there is a space on either side of the direct child selector!

Immediate Adjacent Sibling Selector  +

The immediate adjacent sibling selector is a bit trickier to understand. This selector is represented by a + sign.

When you use this selector, you are targetting the immediate adjacent sibling following an element. Consider the following example:

Example #4

In this example, we have the usual wrapper div element with the ID of #example-4

Within this div wrapper, we have some p elements, some ul elements, and some other nested div elements. The goal here is to target all ul elements that are immediate adjacent siblings following a p element. In other words, if there is a paragraph element followed immediately by an unordered list at the same level of nesting (i.e. the unordered list is a SIBLING of the paragraph, NOT a CHILD), we are going to increase the font-size of the unordered list to 24 pixels and make the font weight bold.

Here is the CSS for the following example:

#example-4 p + ul {
    font-size: 24px;
    font-weight: bold;
}

Do you see what's happening here? Our CSS is saying to change the font size and weight of all ul elements that are immediate, following SIBLINGS of any paragraph element within the #example-4 wrapper div

Therefore, the second unordered list should NOT be styled, since it is not the immediate adjacent sibling to a paragraph element. The third unordered list shouldn't be styled either since it's an immediate adjacent sibling to a div element, not a p element. And finally, the last unordered list should not be styled since it is a CHILD of a div element. Make sense?

Note: FYI, you can't put an unordered list inside of a paragraph. This is invalid HTML and most browsers will automaticaly close the paragraph before the unordered list begins.

Pseudo Selectors

In CSS, we also have things called pseudo selectors which help us even further narrow down / pinpoint elements we want to style. We are going to discuss three of these: first-child, last-child, and nth-child()

The last-child and nth-child() pseudo selectors are not supported in Internet Explorer 8 and below. However, the first-child selector is. Keep this in mind when developing with cross-browser compatibility in mind.

First Child

With the :first-child pseudo selector, we can target the first child of any parent element. What's important to note is that when we use this pseudo selector, we are going to refer to the actual child element we wish to target/style.

Also, when we use pseudo selectors, the syntax is to use a colon right after the selector you're targetting (no space!) followed by the pseudo selector name. Consider the following:

ul li:first-child {
    font-weight: bold;
}

This CSS is saying to target the first child li element in all ul elements and to give it a font-weight of bold. Therefore, it would stand to reason that the first list item of any unordered list would have a bold font weight.

Notice the syntax too... we are applying the :first-child pseudo selector to the <li> element itself, NOT to the ul element.

Nth-Child

And finally, we come to the :nth-child() pseudo selector. This one is pretty cool, as we can pass in different things into the parentheses to achieve different results.

With this pseudo selector, we can pass in the following:

  1. A specific number (1, 3, 8, etc.) that corresponds to the child position (for example, 1 is first-child; in a list of 8 items, 8 would be the last-child)
  2. The words "even" or "odd"
  3. A formula of (an + b) where a is the cycle size, n is a counter (starts at 0), and b is the offset value (ex: 2n + 1)

You can read more about the :nth-child() selector here at CSS Tricks and here at W3Schools

Pseudo Classes

So, you just learned about pseudo selectors. Another type of selector is called a pseudo class and it shares the similar syntax. The most basic example of a pseudo class is the hover state on an anchor tag (an a element). Consider the following CSS:

ul li a:hover {
    color: green;
    font-weight: bold;
    font-size: 20px;
}

This CSS would look for all anchor tags within all li elements within any unordered list. When you hover over the anchor tags, then we change these same anchor tags' styles accordingly. The result of this CSS is as follows:

You'll see terms like pseudo class, pseudo selector, and pseudo element thrown around sometimes seemingly interchangably. While they aren't all the exact same, some terms can refer to the same concept/idea. For example, the :first-child pseudo selector can also be considered a pseudo class. The ::first-letter can be considered a pseudo class or a pseudo element. Semantics and usage are at play here. Don't concern yourself too much with the terminology at this stage. Just understand and practice the concepts.

Pseudo Elements

And finally... pseudo elements!

Pseudo elements can be used to add some cool styles/effects to elements. We can even add content before and after elements with pseudo elements. Pseudo elements use a double-colon in their syntax as opposed to a single colon. Both a double or single colon will work, but the CSS3 spec dictates that a double-colon should be used for pseudo elements. For example, let's look at the ::first-letter pseudo element. Check out the following CSS (and take note of the syntax):

#example-13 p::first-letter {
    font-size: 32px;
    font-weight: bold;
    font-family: serif;
}

With the ::before pseudo element, we can insert and style content before some element. If you aren't aware of the content CSS property, you will be now. We can actually insert content before an element using a combination of the ::before pseudo element and the content CSS property.

    color: green;
}

Note: You might be thinking: "Well, can we insert HTML content before an element using the ::before pseudo element?" Unfortunately, per the spec, you can NOT generate markup via the content CSS property. What about HTML Entities? For example, something like & n b s p ; (the HTML entity code for a space... stands for "non-breaking space"). Yes! We can do this, but we can NOT use the HTML Entity code. We must use the hex code for these. See this article for more information. You can also check out this article as well for more information on the CSS content property.


So, now you know about the ::before pseudo element. Well, we also have the ::after pseudo element as well, and you guessed it--this allows us to insert/style content after an element. One of the best uses for this is clearing floats. You may be aware of using the <div class="clearfix"></div> method to clear your floats, which means you need to insert an empty div after your floats to clear them. With the ::after pseudo element, we can actually clear our floats with no need to add empty div elements anywhere. Consider the following example:

Example #15

I am floated left.
I am also floated left.

The above example contains the usual wrapper div with ID of #example-15 (either Inspect Element on this example in your browser or look at the markup in this index.html file)

Inside of this we have floated elements. We need to clear these floats. We could put an empty div with class of .clearfix after the floated elements to clear them (and of course, we'd have to target .clearfix in our CSS and say clear:both; to do this). However, in this case, I used the ::after pseudo element instead. This is the CSS I used to clear the floats inside of the wrapper div with ID of #example-15:

#example-15::after {
    content: ' ';   /* we MUST have a content property! */
    clear: both;
}

If no content property is defined in a pseudo ::before or ::after element, then no styles will take effect! You must specify a content property, even if it is blank! Use a pair of single quotes or double quotes to set the content equal to a blank string (a programming term).

So, this CSS looks for this #example-15 wrapper div and then right AFTER this element, the CSS employs a clear:both; property to clear the floats. Boom. Done. No empty div elements. No other bogus clearing tricks. Simply a quick line of CSS using the ::after pseudo element.
